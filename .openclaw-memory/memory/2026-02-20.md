## 经验教训

### 1. 关于 Git Push/Pull
- **问题**：AI 有时忘记运行 todo-push.sh 和 todo-push-commit.sh，导致变更留在本地，没有推送到远程仓库
- **解决方案**：
  - 在完成任务后，立即运行 todo-push.sh 来生成变更摘要
  - 生成 commit message 后，立即运行 todo-push-commit.sh 来提交和 push
  - 形成固定的工作流程：完成任务 → 运行 todo-push.sh → 生成 commit message → 运行 todo-push-commit.sh
- **重要性**：确保变更及时推送到远程仓库，多端同步，避免数据丢失

### 2. 关于笔记整理
- **成功经验**：
  - 先在笔记库中广泛搜索，找到最合适的已有的笔记
  - 找到最合适的 section，而不是随便找个地方就放
  - 将内容整合到合适的 section 中，而不是创建新文件
  - 附上来源链接作为引用，用 markdown 格式标注链接
  - 尽量精简语言，精炼整理笔记
  - 格式上：减少不必要的加粗、尽量对齐原文件的格式

### 3. 关于 Playwright 和反爬机制
- **问题**：知乎的反爬机制很强，即使使用 Playwright 模拟浏览器，还是返回了 403 错误
- **下一步**：
