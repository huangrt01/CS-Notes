1. 观点先行
- OpenClaw 不算一个非常“成熟”、“完美”的产品：
  - 名义上开箱即用，但真正用起来需要用户与其进行较长时间的磨合
    - 产品能力的发挥，严重依赖于 Prompt Engineering / 记忆构建，需要用户与其磨合
    - 内置的 skills 较少，关键的基础能力需要用户自行建设，如联网搜索、语音转文字、PDF Parser 等
  - token 消耗大，很容易消耗上亿 token，给用户带来的成本负担较重
    - OpenClaw Token 消耗榜，KIMI 而非最强能力的 Seed-Code 2.0 排第一，因为个人用户较难承受最强能力模型的开销（用户吐槽 OpenClaw 烧 Token）
  - 纯粹的执行任务的能力，远不如最强产品：
    - 代码任务不如 Claude Code、TRAE IDE
    - 【补充】
    - 知识管理不如火山 AI 知识管理
- 但 OpenClaw 是少数做到了“出圈”、用户数 爆发式自增长 的 AI Agent 产品。
  - 产品能力赶上了模型能力的拐点 -> 从而可行：
    - 当下模型的综合能力 & Token 价格达到某个拐点，让大量基于记忆或 Prompt Engineering 的个性化 Agent 能力达标，从而让 OpenClaw “可用”。

OpenClaw 的价值
1. 沙箱自主能力
特点：
- Heartbeat 自驱动机制
- 自发性后台任务执行
- 不需要人工干预，自主推进任务
优点：
- 可以在用户睡觉时自主推进 todo
- 周期性检查任务状态
- 形成闭环自我迭代：执行 → 发现新任务 → 添加新任务 → 继续执行
2. 和各种 IM 的结合

特点：
- 支持多种 IM 渠道（Lark/飞书、Telegram、Slack、Discord、WhatsApp 等）
- 多渠道接入能力
- 统一的 Agent 接口
优点：
- 用户可以通过自己习惯的 IM 渠道与 Agent 交互
- 不需要学习新的工具
- 降低使用门槛

特点：

Agent 未来走向：通用还是个性化？

问题

Agent 未来会走向通用还是个性化？

我的回答

感觉是两个方向：

方向一：通用能力走向原子化，从而每个人都有一套自己的个性化 Agent 系统

特点：
- 通用能力被拆分成原子化的组件
- 每个人可以根据自己的需求组合这些组件
- 形成自己的个性化 Agent 系统
优点：
- 高度灵活
- 每个人都有适合自己的系统
- 可以深度定制
缺点：
- 需要自己组合和配置
- 门槛较高
- 需要一定的技术能力
方向二：一套通用+具备个性化能力的 Agent 系统，服务所有人

特点：
- 一套通用的 Agent 系统
- 内置个性化能力
- 服务所有人
优点：
- 开箱即用
- 门槛低
- 不需要自己配置
缺点：
- 灵活性较低
- 可能无法满足所有人的需求
- 个性化程度有限
具体走向我不懂

思考：
- 具体会走向哪个方向？
- 还是两个方向并存？
- 或者有其他的方向？

与 OpenClaw 的联系

这个问题和 OpenClaw 的使用心得有一定的联系：

- 如果走向方向一（通用能力原子化 + 个性化系统），那么 OpenClaw 这种需要磨合的系统是有价值的
- 如果走向方向二（通用+个性化系统服务所有人），那么 OpenClaw 这种需要磨合的系统可能需要调整
