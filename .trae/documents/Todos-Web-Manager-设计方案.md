# Todos Web Manager 设计方案

## 概述

为 CS-Notes 项目设计一个 Web 端的任务管理器，支持手机提交任务、查看任务列表、标记任务完成等功能。

## 需求分析

### 核心功能

1. **任务提交入口**
   - 手机端可用的简单表单
   - 支持输入任务内容、优先级、标签、链接、截止日期
   - 提交后写入 `.trae/documents/INBOX.md` 或 Pending

2. **任务列表页**
   - 显示所有任务（Pending/进行中/已完成）
   - 支持按状态筛选
   - 支持按优先级筛选
   - 支持搜索功能

3. **任务详情页**
   - 显示任务完整信息
   - 支持标记任务完成
   - 支持编辑任务
   - 支持查看 Plan
   - 支持查看执行产物（摘要/链接/diff/复现命令）

4. **Plan 管理**
   - 支持为任务添加 Plan
   - Plan 包含：目标、假设、改动点、验收标准、风险
   - Plan 审核通过后任务才能进入执行队列

5. **批量操作**
   - 支持批量 kick off Plan
   - 支持批量 review Plan
   - 支持批量标记任务完成

## 技术方案选型

### 方案一：纯静态 HTML + JavaScript（推荐）

**优点：**
- 开发快速，部署简单
- 无需后端服务器
- 可以直接访问本地文件系统（通过浏览器 File API）
- 适合快速原型验证

**缺点：**
- 功能受限
- 无法实现实时同步
- 需要手动刷新

**技术栈：**
- 前端框架：Vue.js（轻量级）或 React
- UI 组件库：Element Plus 或 Ant Design
- 样式：Tailwind CSS

### 方案二：轻量级后端 + 前端

**优点：**
- 功能更强大
- 支持实时同步
- 可以集成更多功能

**缺点：**
- 开发复杂度较高
- 需要部署后端服务器
- 需要考虑安全性

**技术栈：**
- 后端：Python Flask 或 FastAPI
- 前端：Vue.js 或 React
- 数据库：SQLite（轻量级）

### 方案三：基于现有工具

**优点：**
- 无需开发
- 可以直接使用

**缺点：**
- 可能不符合需求
- 需要定制化

**可选工具：**
- Todoist
- Trello
- Notion
- GitHub Issues

## 推荐方案

**推荐方案一（纯静态 HTML + JavaScript）**，原因：
1. 快速验证需求
2. 部署简单
3. 可以先跑通核心流程
4. 后续可以升级到方案二

## 需要用户决策的问题

1. **技术方案选择**：你希望使用哪个方案？
   - 方案一：纯静态 HTML + JavaScript（快速原型）
   - 方案二：轻量级后端 + 前端（功能更强大）
   - 方案三：基于现有工具（无需开发）

2. **UI/UX 偏好**：你希望 Web Manager 的风格是什么样的？
   - 简洁实用（类似 GitHub Issues）
   - 现代化（类似 Notion）
   - 手机端优先（类似 Todoist）

3. **功能优先级**：哪些功能是必须的，哪些是可选的？
   - 必须：任务提交、任务列表、标记完成
   - 可选：Plan 管理、批量操作、搜索筛选

4. **部署方式**：你希望如何部署这个 Web Manager？
   - 本地运行（访问 localhost）
   - 部署到火山引擎（公网访问）
   - 部署到 GitHub Pages（静态网站）
