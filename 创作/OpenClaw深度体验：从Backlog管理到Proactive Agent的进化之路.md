# OpenClaw 深度体验：从 Backlog 管理到 Proactive Agent 的进化之路

接入 OpenClaw 的 9 天半时间里，观察到一个清晰的现象：当 AI 助手从被动响应转向主动推进时，知识管理和任务执行的效率会出现质的飞跃。

本文将结合 9 天半的实战数据与深度思考，探讨这一“不完美”但“具有突破性”的 Proactive Agent 形态。

## 观点先行

首先需要明确的是，**OpenClaw 目前并不算一个非常“成熟”、“完美”的产品**：

*   **磨合成本高**：名义上开箱即用，但真正发挥效能严重依赖 Prompt Engineering 和记忆构建，用户需要与其进行较长时间的磨合。
*   **基础能力待建**：内置 Skills 较少，关键的基础能力（如联网搜索、语音转文字、PDF Parser）需要用户自行建设。
*   **Token 消耗大**：很容易消耗上亿 Token，个人用户较难承受最强能力模型（如 Seed-Code 2.0）的开销。
*   **纯代码能力差距**：在纯粹的代码执行任务上，目前仍不如 Claude Code 或 Trae IDE 等最强产品。

**但 OpenClaw 是少数做到了“出圈”、实现用户爆发式自增长的 AI Agent 产品。**

其核心在于它赶上了模型能力的拐点，并初次具备了完整的“五感与心智”：
*   **眼**：多模态交互能力。
*   **耳/口**：各类 IM + Web Manager
*   **脑**：LLM 的推理规划能力。
*   **手**：增强的 Tool/Skill 调用能力。
*   **心**：**Heartbeat（心跳）设计**，赋予了产品自主性，使其初步具备了 Proactive Agent 的雏形。

这种“能对话、后台自主干活的机器人”在大众视角里是有趣且具有突破性的，也标志着「Agent 代理人/企业消耗 Token」的时代初见端倪。

## 核心理念：基于 Backlog 的 Proactive Agent

在 9 天半的体验中，我尝试探索一种全新的协作模式：**基于 Backlog（待办事项）的 Proactive Agent**。

这种模式的核心价值在于：

1.  **沙箱自主能力（Sandbox Autonomy）**
    *   **Heartbeat 自驱动**：利用心跳机制，Agent 可以在后台周期性检查任务状态。
    *   **不阻塞交互**：不需要人工实时干预，Agent 可以在用户睡觉时自主推进 Todo。
    *   **闭环自我迭代**：执行任务 → 发现新任务 → 添加新任务 → 继续执行。

2.  **全渠道接入（Omni-channel）**
    *   支持各种 IM（Lark/Telegram/Slack 等），让用户可以在自己习惯的渠道与 Agent 交互，降低使用门槛。

这不仅仅是一个工具，更像是一个**不知疲倦、自主推进的数字员工**。为了验证这一理念，我进行了一次为期 9 天半的深度接入实验。

## 实战数据与观察

从 2026-02-16 到 2026-02-26，共 9 天半时间，OpenClaw 在本仓库的接入数据如下：

*   **总 commit 数**：366 个（平均每天约 38.5 个）
*   **笔记整理次数**：30 次（平均每天约 3.16 次）
*   **核心功能实现**：5 个核心系统 + 20+ 技能集成
*   **技能集成数量**：20+ 个 Skills 集成

**Commit 趋势呈现明显的三个阶段**：
1.  **初期（02-16 到 02-18）**：快速适应，主要是基础配置和规则建立。
2.  **中期（02-19 到 02-22）**：快速迭代，Commit 数量激增，主要是功能实现和优化。
3.  **后期（02-23 到 02-26）**：稳定优化，主要是 Bug 修复和体验优化。

这些数据证明了：一旦度过磨合期，Proactive Agent 能带来巨大的工程产出。

## 核心功能实现

为了支撑上述理念，我们在 OpenClaw 上构建了以下核心系统：

### 1. Todos Web Manager：单一数据源
实现了完整的任务管理界面（Pending/In-progress/Completed）。其核心设计原则是**单一数据源（Single Source of Truth）**，以 JSON 为唯一数据源，避免了多端协作的数据不一致问题。

### 2. Task Execution Logger：可观测闭环
**让 AI 的工作过程不再是黑盒**。实现了完整的任务状态机（Planning → Executing → Completed → Failed），记录详细的执行日志和指标（时间、重试次数、成功率）。

### 3. Plan Generator + Hybrid Executor
将 Plan 机制本质化重构：Plan 不再是独立的状态，而是 Todo 的一个属性。支持“智能拆解 + 用户审核”的混合执行方式，平衡效率与可控性。

### 4. Commit Diff 展示
在完成 Todo 时展示修改前后的 Commit Diff。这个看似微小的功能，**让任务交付更加透明，让用户能够快速验证变更是否符合预期**。

### 5. 20+ Skills 生态集成
集成了文档处理（PDF/Markdown）、网页抓取（Playwright/Search）、Todo 管理、效率工具（Whisper/Image Gen）等 20+ 个技能，构建了丰富的功能生态。

## 关键的人类经验

接入 OpenClaw 的过程中，沉淀了 9 个关键的人类经验。这些经验比技术实现本身更有价值，因为它们代表了人与 Proactive Agent 协作的最佳实践。

### 1. 需要磨合（Friction is Necessary）
OpenClaw 不是开箱即用的。初期需要配置、调整，积累适合自己的工作流。
*   **启示**：不要期望即插即用。经过磨合后的 Agent 能深度理解你的意图，效率会呈指数级上升。但同时也带来一个隐忧：**这种高度定制化的磨合能力，在切换到新项目时复用性如何？**这是一个值得持续探索的问题。

### 2. 安全意识：绝对防线
**惨痛教训**：曾差点将 API Key 或公司项目内容提交到公开仓库。
*   **启示**：必须建立严格的白名单（仅允许 `Notes/` 等）和黑名单（禁止 `公司项目/`）机制。在写入任何文件前，必须进行敏感内容检查。

### 3. Git 操作 SOP化
Agent 的 Git 操作必须有标准流程（SOP），不能直接运行 Git 命令。
*   **启示**：使用脚本（`todo-push.sh`）封装 Git 操作，强制执行状态检查、白名单验证和 Commit Message 规范。完成 Todo 后**立刻 Push 并反馈 Commit 链接**，不要等待用户询问。

### 4. 笔记整理 SOP
防止 Agent 在整理笔记时产生幻觉或丢失信息。
*   **启示**：先提取目录结构，再寻找最合适的位置整合。**严格遵循“无幻觉”原则**，所有外部内容必须附带来源链接。

### 5. Todo 管理原则
以 JSON 为单一数据源。
*   **启示**：Plan 是 Todo 的属性。任务归档前必须增加用户 Review 流程。Review 不通过则回退状态并附带意见。

### 6. 用户交互原则
**用户体验优先**。无论 AI 正在执行什么后台任务，收到用户消息必须**立即回复**。
*   **启示**：耗时任务必须后台化（Background Mode）。定期（每10分钟）主动汇报进度。只有在需要用户做“选择题”时才打断用户，否则自主推进。

### 7. 任务交付原则
**交付 ≠ 代码实现**。
*   **启示**：交付的标准是“落地”——成为仓库的默认设置，并经过端到端的体验验证。不要只交付“玩具代码”。

### 8. 错误反思原则
失败是宝贵的资产。
*   **启示**：遇到失败必须在文档中记录错误的命令和原因，避免重复犯错。

### 9. Review 瓶颈与缓解路径
**现象**：Review AI 的工作成为了当前的效率瓶颈（14 次 Review 不通过）。
*   **启示**：这是初期磨合的必经之路。随着 Agent 个性化能力的提升（更懂你的偏好），Review 的通过率会逐渐提高，瓶颈将自行缓解。

## 未来展望：通用还是个性化？

站在 9 天半的节点上展望未来，Agent 的发展似乎面临两个方向的抉择：

1.  **方向一：通用能力原子化 + 个性化系统**
    *   通用能力被拆分为原子组件，每个人根据需求组装自己的 Agent。
    *   **优点**：高度灵活，深度定制。
    *   **OpenClaw 的位置**：目前更符合这一方向，通过高磨合成本换取高个性化价值。

2.  **方向二：通用系统 + 内置个性化能力**
    *   一套通用的强大系统，内置极强的自适应个性化能力，服务所有人。
    *   **优点**：开箱即用，门槛低。
    *   **趋势**：可能是大模型厂商（如 Claude/OpenAI）最终追求的终局。

无论走向何方，这 9 天半的经历都标志着一个转折点：**我们正在从“使用工具”的时代，迈向“与数字员工共事”的时代。**

在这个过程中，保持理性，建立规则，并在磨合中共同进化，或许是我们面对 AI 浪潮最好的姿态。
